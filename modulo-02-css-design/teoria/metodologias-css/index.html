BEM - Block Element Modifier

Metodología que nos ayuda a crear componentes CSS reutilizables y mantenibles con una nomenclatura clara y predecible.

🎯 ¿Qué es BEM?

BEM es una metodología de nomenclatura que divide los componentes en:

  • Block (Bloque): Componente independiente → .producto
  • Element (Elemento): Parte del bloque → .producto__titulo
  • Modifier (Modificador): Variación del bloque/elemento → .producto--destacado
.producto { } .producto__titulo { } .producto__descripcion { } .producto__boton { } .producto--destacado { } .producto__boton--primario { }

✅ Ventajas:

  • Nomenclatura clara y predecible
  • Evita conflictos de especificidad
  • Componentes verdaderamente reutilizables
  • Fácil mantenimiento y debugging

❌ Desventajas:

  • Nombres de clases muy largos
  • HTML puede verse verboso
  • Curva de aprendizaje inicial

📋 Cuándo usar BEM:

  • Proyectos grandes con múltiples desarrolladores
  • Cuando necesitas componentes reutilizables
  • Aplicaciones que requieren escalabilidad
  • Cuando trabajas con frameworks/librerías

Metodología de Módulos

Organización del CSS en módulos independientes y reutilizables, cada uno con su propia responsabilidad y función específica.

🧩 ¿Qué son los Módulos CSS?

Los módulos son componentes CSS independientes que encapsulan estilos específicos para una funcionalidad concreta.

/* MÓDULO: Navegación */ .navbar { } .navbar-container { } .navbar-logo { } .navbar-menu { } .navbar-link { } /* MÓDULO: Modal */ .modal { } .modal-content { } .modal-header { } .modal-body { } .modal-footer { }

✅ Ventajas:

  • Organización clara por funcionalidad
  • Fácil localización y mantenimiento
  • Reutilización eficiente de código
  • Desarrollo en equipo más organizado
  • Testing más sencillo por módulo

❌ Desventajas:

  • Puede generar muchos archivos CSS
  • Requiere estructura de carpetas organizada
  • Necesita bundler para optimizar en producción
  • Curva de aprendizaje para organización

📋 Cuándo usar Módulos:

  • Proyectos medianos a grandes
  • Cuando trabajas con preprocesadores (Sass, Less)
  • Equipos que necesitan organización clara
  • Aplicaciones con muchos componentes
  • Cuando quieres reutilizar componentes

Metodología de Utilidades

Clases pequeñas y específicas que aplican un único estilo, permitiendo construir interfaces combinando múltiples utilidades.

⚡ ¿Qué son las Utilidades CSS?

Las utilidades son clases de propósito único que aplican un estilo específico y se combinan para crear diseños complejos.

/* Ejemplos de utilidades */ .u-text-center { text-align: center !important; } .u-mb-3 { margin-bottom: 24px !important; } .u-flex { display: flex !important; } .u-justify-between { justify-content: space-between !important; } .u-bg-primary { background-color: #3498db !important; } .u-text-white { color: white !important; }

✅ Ventajas:

  • Desarrollo extremadamente rápido
  • Consistencia visual automática
  • Tamaño de CSS predecible
  • Ideal para prototipado
  • No necesitas inventar nombres de clase

❌ Desventajas:

  • HTML puede verse desordenado
  • Muchas clases por elemento
  • Curva de aprendizaje para memorizar clases
  • Puede ser difícil mantener en proyectos grandes

📋 Cuándo usar Utilidades:

  • Prototipado rápido de interfaces
  • Proyectos pequeños a medianos
  • Cuando necesitas velocidad de desarrollo
  • Equipos que conocen bien la metodología
  • Para ajustes rápidos y específicos
