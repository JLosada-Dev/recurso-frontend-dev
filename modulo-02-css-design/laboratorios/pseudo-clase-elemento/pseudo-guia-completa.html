Pseudo-clases y Pseudo-elementos en CSS

Guía completa con ejemplos prácticos y buenas prácticas

1. Pseudo-clases

Las pseudo-clases permiten seleccionar elementos basándose en su estado o posición. Se escriben con un solo dos puntos (:).

Estados de Interacción

Uso: :hover, :focus, :active
Cuándo usar: Para mejorar la experiencia del usuario y proporcionar feedback visual

Pseudo-clases Estructurales

  • Primer elemento
  • Segundo elemento
  • Tercer elemento
  • Cuarto elemento
  • Quinto elemento
  • Último elemento

Uso: :first-child, :last-child, :nth-child(n), :nth-of-type(n)
Cuándo usar: Para estilizar elementos específicos sin añadir clases HTML

Estados de Formulario

Uso: :valid, :invalid, :required, :checked, :disabled
Cuándo usar: Para validación visual y mejorar la UX en formularios

2. Pseudo-elementos

Los pseudo-elementos permiten estilizar partes específicas de un elemento o crear contenido virtual. Se escriben con doble dos puntos (::).

4. Mejores Prácticas y Recomendaciones

✅ Sí hacer

  • Usar :focus para accesibilidad
  • Combinar :hover con transiciones
  • Usar ::before/::after para decoraciones
  • Aplicar :nth-child para patrones
  • Usar ::selection para consistencia

❌ No hacer

  • Abusar de pseudo-elementos complejos
  • Olvidar estados de focus
  • Usar solo :hover sin :focus
  • Crear layouts con pseudo-elementos
  • Ignorar la especificidad

💡 Consejo de Rendimiento

Los pseudo-elementos ::before y ::after son elementos reales en el DOM. Úsalos con moderación en listas muy largas para mantener buen rendimiento.
