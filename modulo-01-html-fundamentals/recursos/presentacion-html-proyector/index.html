¿Qué es HTML?

HyperText Markup Language
(Lenguaje de Marcado de Hipertexto)

Propósito: Dar estructura y significado al contenido web mediante etiquetas

¿Cómo funciona la web?

  1. Usuario escribe URL en navegador
  2. Navegador hace petición HTTP/HTTPS al servidor
  3. Servidor responde con documentos (HTML, CSS, JS)
  4. Navegador interpreta HTML y muestra página web
URL: Localizador único de recursos en línea
HTTP/HTTPS: Protocolo de transferencia de datos
HTML: Lenguaje de estructura y contenido

Configuración del Entorno

Editor de Código

  • Visual Studio Code
  • Sublime Text
  • Atom

Características: Coloreado de sintaxis, autocompletado, extensiones

Navegador Web

  • Chrome
  • Firefox
  • Edge
  • Safari

Uso: Visualizar y probar código

Servidor Local (opcional)

  • XAMPP
  • MAMP
  • Live Server

Uso: Simular entorno real

Estructura de Carpetas

proyecto/
├── index.html    # archivo principal
├── css/          # hojas de estilo
├── js/           # scripts JavaScript
└── img/          # imágenes

Anatomía de un Documento HTML

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <title>Mi página</title>
  </head>
  <body>
    <h1>Hola mundo</h1>
    <p>Mi primer sitio web</p>
  </body>
</html>
<!DOCTYPE html> - Declara HTML5
<html> - Elemento raíz del documento
<head> - Metainformación no visible
<body> - Contenido visible de la página

Etiquetas, Elementos y Atributos

Etiqueta: Palabra clave entre < > que define función
Elemento: Etiqueta apertura + contenido + etiqueta cierre
Atributo: Información extra que modifica comportamiento

Ejemplos

Elemento Normal

<p>Hola mundo</p>

Con Atributos

<img src="foto.jpg" alt="Descripción">

Elementos Vacíos

<br>, <hr>, <input>

Anidación

<section>
  <h2>Título</h2>
  <p>Párrafo</p>
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
  </ul>
</section>

HTML Semántico - ¿Por qué?

Accesibilidad mejorada

Lectores de pantalla navegan mejor

Mejor SEO

Motores de búsqueda entienden contenido

Código mantenible

Más fácil de leer y mantener

Evitar <div>

Solo usar cuando no hay etiqueta específica

Estructura Básica

<body>
  <header><!-- Cabecera --></header>
  <main><!-- Contenido principal --></main>
  <aside><!-- Contenido complementario --></aside>
  <footer><!-- Pie de página --></footer>
</body>

Etiquetas Semánticas Principales

Etiqueta Función Uso
<header> Cabecera de página/sección Logo, título, navegación
<nav> Zona de navegación Menús, breadcrumbs, paginación
<main> Contenido principal Tema central (único por página)
<section> Sección temática Capítulos, bloques temáticos
<article> Contenido independiente Noticias, posts, productos
<aside> Contenido complementario Publicidad, widgets, enlaces
<footer> Pie de página Copyright, contacto, redes

Títulos y Texto

Concepto: Las etiquetas de texto son fundamentales para estructurar el contenido de manera jerárquica y semánticamente correcta.

Encabezados

Jerarquía

<h1>Título Principal</h1>
<h2>Subtítulo</h2>
<h3>Sección específica</h3>

Reglas

  • Solo un <h1> por página
  • Seguir orden jerárquico
  • No saltar niveles

Elementos de Texto

<p> - Párrafos

<p>Este es un párrafo.</p>

<strong> - Importante

<p>Esto es <strong>importante</strong>.</p>

<em> - Énfasis

<p>Esto está <em>enfatizado</em>.</p>

<mark> - Resaltado

<p><mark>Texto resaltado</mark></p>

Listas - Organización del Contenido

Concepto: Las listas permiten organizar información de manera estructurada y accesible.

Lista Desordenada (<ul>)

<ul>
  <li>Primer elemento</li>
  <li>Segundo elemento</li>
  <li>Tercer elemento</li>
</ul>

Uso: Elementos sin orden específico

Lista Ordenada (<ol>)

<ol>
  <li>Primer paso</li>
  <li>Segundo paso</li>
  <li>Tercer paso</li>
</ol>

Uso: Secuencias específicas

Lista de Definiciones (<dl>)

<dl>
  <dt>HTML</dt>
  <dd>Lenguaje de marcado</dd>
  <dt>CSS</dt>
  <dd>Hojas de estilo</dd>
</dl>

Uso: Pares término-definición

Listas Anidadas

<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>

Enlaces y Multimedia Básica

Enlaces (<a>)

Enlace Externo

<a href="https://ejemplo.com" target="_blank">
  Visitar sitio
</a>

Enlace Interno

<a href="#seccion">Ir a sección</a>

Enlace de Email

<a href="mailto:contacto@ejemplo.com">
  Enviar email
</a>

Imágenes (<img>)

Imagen Básica

<img src="imagen.jpg" alt="Descripción" 
     width="300" loading="lazy">

Con Contenedor Semántico

<figure>
  <img src="grafico.jpg" alt="Gráfico">
  <figcaption>Evolución de ventas</figcaption>
</figure>

Buena práctica: Siempre incluir atributo alt en imágenes para accesibilidad

Formularios - Interacción con Usuarios

Estructura del Formulario

<form action="/procesar" method="post">
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre" required>
  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  
  <button type="submit">Enviar</button>
</form>

Tipos de Input

Campos de Texto

  • type="text" - Texto simple
  • type="email" - Correo electrónico
  • type="password" - Contraseña
  • type="number" - Números

Opciones

  • type="checkbox" - Opción múltiple
  • type="radio" - Opción única
  • <select> - Lista desplegable

Otros Elementos

  • <textarea> - Texto multilínea
  • <button> - Botones
  • type="file" - Subir archivos

Multimedia Avanzada

Video (<video>)

<video controls width="640" height="480">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Tu navegador no soporta video.
</video>

Audio (<audio>)

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Tu navegador no soporta audio.
</audio>

Contenido Externo (<iframe>)

<iframe src="https://www.youtube.com/embed/VIDEO_ID" 
        width="560" height="315" 
        title="Video tutorial"></iframe>

Atributos Útiles

  • controls - Controles de reproducción
  • autoplay - Reproducción automática
  • loop - Reproducción en bucle
  • muted - Sin sonido

Formatos Comunes

  • Video: MP4, WebM, OGV
  • Audio: MP3, OGG, WAV

Datos Tabulares

Estructura de Tabla

<table>
  <thead>
    <tr>
      <th>Producto</th>
      <th>Precio</th>
      <th>Stock</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Laptop</td>
      <td>$999.00</td>
      <td>15</td>
    </tr>
  </tbody>
</table>

Elementos de Tabla

  • <table> - Contenedor principal
  • <thead> - Encabezado
  • <tbody> - Cuerpo con datos
  • <tr> - Fila
  • <th> - Celda encabezado
  • <td> - Celda de datos

Atributos Útiles

  • colspan - Ocupa múltiples columnas
  • rowspan - Ocupa múltiples filas
  • scope - Define alcance del encabezado

Cuándo usar: Datos con relación fila-columna, reportes, horarios

Elementos Autocerrables

Concepto: Elementos que no necesitan etiqueta de cierre porque no contienen contenido.

<img> - Imágenes

<img src="foto.jpg" alt="Descripción">

Requeridos: src, alt

<input> - Campos de formulario

<input type="text" name="nombre" required>

Tipos: text, email, password, checkbox, radio

<br> - Salto de línea

Primera línea<br>Segunda línea

Uso: Solo para saltos semánticamente necesarios

<hr> - Separador temático

<p>Párrafo 1</p><hr><p>Párrafo 2</p>

Uso: Cambio de tema o sección

<meta> - Metadatos

<meta charset="UTF-8">
<meta name="viewport" content="width=device-width">

<link> - Recursos externos

<link rel="stylesheet" href="style.css">
<link rel="icon" href="favicon.ico">

Ejemplo Práctico Completo

Descripción: Sitio web que integra todas las etiquetas y conceptos aprendidos

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Portafolio - Desarrollador Web</title>
</head>
<body>
    <header>
        <img src="logo.png" alt="Logo personal" width="80">
        <h1>Juan Pérez</h1>
        <nav>
            <ul>
                <li><a href="#about">Sobre mí</a></li>
                <li><a href="#projects">Proyectos</a></li>
                <li><a href="#contact">Contacto</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section id="about">
            <article>
                <h2>Sobre mí</h2>
                <p>Soy un <strong>desarrollador web</strong> 
                   con experiencia en tecnologías modernas.</p>
            </article>
        </section>

        <section id="contact">
            <h2>Contacto</h2>
            <form action="/contacto" method="post">
                <label for="nombre">Nombre:</label>
                <input type="text" id="nombre" required>
                
                <label for="email">Email:</label>
                <input type="email" id="email" required>
                
                <button type="submit">Enviar</button>
            </form>
        </section>
    </main>

    <footer>
        <p>&copy; 2025 Juan Pérez</p>
    </footer>
</body>
</html>

Buenas Prácticas

Estructura y Semántica

  • Usar HTML semántico antes que <div> genéricos
  • Mantener jerarquía de títulos (un <h1> por página)
  • Un solo <main> por página
  • Estructura lógica sin CSS

Accesibilidad

  • Incluir atributo alt en todas las imágenes
  • Asociar labels con campos de formulario
  • Usar atributos ARIA cuando sea necesario
  • Mantener contraste adecuado

SEO y Rendimiento

  • Title descriptivo y único por página
  • Meta description relevante
  • Atributo lang en <html>
  • Optimizar imágenes para web

Mantenimiento

  • Comentarios explicativos
  • Indentación consistente
  • Validación regular del código
  • Nombres descriptivos

Herramientas de Validación: W3C Markup Validator, WAVE Web Accessibility, Lighthouse
